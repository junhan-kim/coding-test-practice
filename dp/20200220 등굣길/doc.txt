[문제] https://programmers.co.kr/learn/courses/30/lessons/42898
[유형] dp
[풀이]
 최단거리라고 했으나 결국 도착지점에서 시작지점으로 왼쪽or위쪽으로만 이동하면 무조건 최단거리가 된다.
 점화식 : func(x-1,y) + func(x,y-1)   // 해당 위치까지 도달하는데의 최단 거리
 예외 : x<0 || y<0  -> 범위 초과(이미 범위를 초과한 시점에서 최단거리가 될 수 없음.. 뒤로 다시 나오던, 돌아가던 이미 최단거리+1이 되어버림
        (x,y)가 puddle에 있는 경우 -> 위와 같음(이미 최단거리 초과)
 도달한 경우에는 1을 올려주고, 도달하지 못한 경우(예외)에는 0을 올려주어서 경우의 수를 세어준다.
[주요 기능]
[에러]
 1. 계산결과에 MOD 적용 X -> (A+B)%N = (A%N+B%N)%N  // 점화식에 적용할 것.  최종결과에만 적용하면 오버플로우됨
 2. 인덱스와 실제값 처리가 다름 -> puddle은 실제값으로 주어지는데, 인덱스로 착각함.
